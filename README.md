## Краткая теория
Допустим, у вас имеется некая цепь с некой передаточной функцией. Для более простого анализа данной цепи, она разбивается на так называемые звенья. Каждое звено имеет свою заранее известную логарифмическую характеристику и заранее известную передаточную функцию. Деление на звенья происходит довольно простым образом - ПФ цепи раскладывается на множители.

> [!TIP]
> Каждое звено имеет свои амплитудные и фазовые характеристики. Суммарные ЛАХ и ЛФХ получаются при сложении характеристик каждого из звеньев (если звенья стоят последовательно, то есть передаточные функции каждого из звеньев, перемножаются друг на друга и дают передаточную функцию всей цепи).
> Как пример - ПФ цепи K/jw может разложиться на последовательное усилительное (K) и интегрирующее (1/jw) звенья.

Итак, у нас есть 6 типов звеньев:
1. **Усилительное звено** (по факту просто константа)
![Ampl](https://github.com/russkiylis/ZvenoLH/blob/main/images/Ampl.png?raw=true)
2. **Интегрирующее звено** (Линия сверху вниз)
![Integr](https://github.com/russkiylis/ZvenoLH/blob/main/images/Integr.png?raw=true)
3. **Дифференциирующе звено** (Линия снизу вверх)
![Diff](https://github.com/russkiylis/ZvenoLH/blob/main/images/diff.png?raw=true)
4. **Апериодическое звено**  (Сначала 0, затем линия вниз)
![Aperiod](https://github.com/russkiylis/ZvenoLH/blob/main/images/aperiod.png?raw=true)
5. **Форсирующее звено**  (Сначала 0, затем линия вверх)
![Fors](https://github.com/russkiylis/ZvenoLH/blob/main/images/fors.png?raw=true)
6. **Колебательное звено** (по факту, состоит из других звеньев, в программе не реализовано)
![Koleb](https://github.com/russkiylis/ZvenoLH/blob/main/images/koleb.png?raw=true)

## Контрольная работа №1

> [!NOTE]
> Скорее всего ваша первоначальная задача будет состоять в следующем:
> 1. Разложить ПФ цепи на ПФ звеньев, помноженные друг на друга
> 2. Найти ЛХ каждого из звеньев
> 3. Сложить ЛХ и получить ЛХ цепи

Что придётся сделать на контрольной? Разберём по пунктам.

### 1. Нарисовать ЛХ.
В целом, теория про звенья написана сверху. Если кратко, то данную вам ПФ надо разбить на звенья, мысленно (или пунктиром) нарисовать их ЛХ, а затем сложить, получив ЛХ всей цепи. Добавлю ещё картинку.
![ksusha_LH](https://github.com/russkiylis/ZvenoLH/blob/main/images/ksusha_LH.jpg)
> [!CAUTION]
> #### АПДЕЙТ ПОСЛЕ КР 1
> Оказывается, надо рисовать ещё и ЛФХ (годограф). Никто в этом нормально не разобрался, однако во время контрольной я быстро сделал файл [GODOGRAF.m](/GODOGRAF.m), который выводит нечто похожее на годограф. Вместо моей передаточной функции вам надо будет написать туда свою, при этом заменив `p` на `jw`. Если не знаете, как это сделать, скормите нейросети этот файл и вашу передаточную функцию, а лучше потренируйтесь заранее.

### 2. Добавить коррекцию
Коррекцию надо добавлять, если на частоте, на которой ЛАХ пересекла уровень нуля, ЛФХ находится ниже -180 градусов, поскольку для устойчвивости необходим хоть какой-то запас по фазе, который считается как `γ=180-ЛФХ`. Обычно можно добавить форсирующие звенья, однако, так как они не реализуемы в реальности, их заменяют эквивалентом в виде шунта на интеграторе. Для постройки ЛХ цепи с коррекцией в случае, показанном ниже, достаточно добавить в программу два форсирующих звена с `T=4`.
![ksusha_korrec](https://github.com/russkiylis/ZvenoLH/blob/main/images/ksusha_korrec.jpg)
Обратите внимание на пункт 2.
![preobr](https://github.com/russkiylis/ZvenoLH/blob/main/images/preobr.jpg)

### 3. Посчитать матрицы
В данном пункте надо нарисовать схемку с общей обратной связью. Апериодическое звено для удобства можно преобразовать. Далее нумеруем `y(t)` справа на лево перед каждым интегратором. Перед интегратором, логично, ставится `dy(t)/dt`.
Далее пишем дифференциальные уравнения. Для этого смотрим, чем формируются каждые производные (см. картинку). Из этого уравнения делаем переходную матрицу `F`. Далее формируем матрицу наблюдения. Для этого достаточно посмотреть, как формируется окончательный сигнал `y(t)`. В случае на картинке он формируется только сигналом `y1`. И последняя матрица - матрица преобразования входного сигнала `g(t)`. Для её формирования достаточно посмотреть, в каком уравнениии и с каким коэффициентом присутствует `g(t)`.
![ksusha_matricy](https://github.com/russkiylis/ZvenoLH/blob/main/images/ksusha_matricy.jpg)
Обратите внимание на пункт 1.
![preobr](https://github.com/russkiylis/ZvenoLH/blob/main/images/preobr.jpg)

### 4. Посчитать ошибки
для подсчёта ошибок смотрим на ПФ без общей обратной связи, т.е. разомкнутую. Дам пару комментариев. В первой таблице (типовые значения *динамической* ошибки СУ) первый столбец - вид воздействия. смотрим, какого рода воздействие дано. `s` - порядок задающего воздействия. `r` - порядок астатизма. `Nф` - порядок формирующего фильтра. С помощью `r` и `s` надо подобрать нужную формулу. `K` - это вроде как сама ПФ. Вторая таблица - *Флуктуационные ошибки*. В ней надо по первому столбцу подобрать вид ПФ, максимально близкий к тому, что нам дано, и по табличке высчитать всё остальное.
![tablica_oshibok](https://github.com/russkiylis/ZvenoLH/blob/main/images/tablica_oshibok.jpg)

### 5. Посчитать характеристики переходного процесса
Довольно простой пункт. Во первых, по запасу устойчивости определяем вид переходного процесса (малоколебательный, сильноколебательный, монотонный). Далее с помощью запаса устойчивости определяем величину перерегулирования (выброс). А затем по частоте среза (частоте, где ЛАХ пересекается с нулём) находим время нарастания переходного процесса.
![pereh_process](https://github.com/russkiylis/ZvenoLH/blob/main/images/pereh_process.jpg)

1. Допустим, у тебя есть ПФ в виде дроби: `W(p) = 10(1 + 0.2p) / p(1 + 0.5p)`. Для начала переводим это в формат звеньев. Видим, что тут есть
  - Усилительное звено `10`
  - Интегрирующее звено `1/p`
  - Апериодическое звено `1/(1+0.5p)`
  - Форсирующее звено `(1+0.2p)`.

> [!CAUTION]
> #### АПДЕЙТ ПОСЛЕ КР 1
> Раньше при засовывании в усилительное звено 10, программа строила усилитель на 10 дБ, что неверно, так как в ПФ усиление не в дБ. То есть, ранее для корректной работы вам самим нужно было перевести усиление в ПФ в логарифмическй масштаб по формуле `KдБ = 20log10(K)`, а уже затем скармливать программе. Теперь этого делать не надо.

---
2. Создаём прямо в папке новый [Matlab](https://yandex.ru/search/?text=скачать+матлаб+2025)-файл (Ниже я буду описывать файл [ExampleFromReadme.m](/ExampleFromReadme.m)).
> [!WARNING]
> Новый файл должен быть в корневой папке, то есть в той папке, куда вы скопировали репозиторий, то есть лежать рядом с файлом [ZvenoLH.m](/ZvenoLH.m).

В новом файле расписываем звенья в следующем формате:

```
K = 10;
Ta = 0.5;
Tf = 0.2;

W = {
    struct('Type',"Усил",'K',K),...
    struct('Type',"Интегр"),...
    struct('Type',"Апериод",'T',Ta),...
    struct('Type',"Форс",'T',Tf)
};
```

---
3. Создаём объект, в котором будут созданы логарифмические характеристики:

```
LH = ZvenoLH(W);
```
Теперь у нас есть логарифмические характеристики.

---
4. Время вывести логарифмические характеристики. Это можно сделать вручную, как расписано в [ManualExample.m](/ManualExample.m), если вам нужна большая гибкость в отображении графиков. Если же вам не хочется в этом разбираться, можно вывести суммарную ЛХ цепи одной командой:
```
LH.showSumLH(xlimit, ylimitL, ylimitPhi);  % для вывода ЛХ всей цепи
```
`xlimit` - границы графиков по оси Х. Рекомендую для начала ставить `[1/32 32]`, а затем подгонять.
`ylimitL` - границы графика ЛАХ по оси У. Рекомендую для начала ставить `[-20 20]`.
`ylimitPhi` - границы графика ЛФХ по оси У. Рекомендую для начала ставить `[-150 150]`.

Для вывода ЛХ отдельных звеньев можно воспользоваться этим:
```
LH.showZvenoLH(index, xlimit, ylimitL, ylimitPhi);  % для вывода ЛХ отдельного звена.
```
`index` - порядковый номер звена, например в нашем случае первое звено - усилительное, второе - интегрирующее, и т.д.

> Открытый файл (то есть тот файл, код которого вы видите) запускается кнопочкой Run во вкладке Editor!
